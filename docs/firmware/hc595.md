# Driver 74HC595

## Initialisation SPI

Le 74HC595 utilise **SPI3_HOST** (bus séparé du TLC5940 sur SPI2) :

```c
spi_bus_config_t bus_595 = {
    .miso_io_num   = -1,
    .mosi_io_num   = REG_MOSI,   // GPIO13
    .sclk_io_num   = REG_SCLK,   // GPIO14
    .max_transfer_sz = 64
};
spi_device_interface_config_t dev_595 = {
    .clock_speed_hz = 5000000,
    .mode           = 0,
    .spics_io_num   = -1,
    .queue_size     = 7
};
spi_bus_initialize(SPI3_HOST, &bus_595, SPI_DMA_CH_AUTO);
spi_bus_add_device(SPI3_HOST, &dev_595, &spi_595);
```

## Sélection d'un plan

```c
static void select_plane(int p) {
    // mask : bit p à 0 (MOSFET passant), tous les autres à 1
    uint16_t mask  = ~(1 << p);
    uint8_t data[2] = { mask >> 8, mask & 0xFF };

    spi_transaction_t t = { .length = 16, .tx_buffer = data };
    spi_device_transmit(spi_595, &t);

    // Impulsion STCP → verrouillage dans le latch output
    gpio_set_level(REG_STCP, 1);
    esp_rom_delay_us(1);
    gpio_set_level(REG_STCP, 0);
}
```

### Table des valeurs mask

| Plan actif | mask (hex) | Signification |
|-----------|-----------|---------------|
| 0 | 0xFFFE | Q0=0, Q1..Q9=1 |
| 1 | 0xFFFD | Q1=0 |
| 5 | 0xFFDF | Q5=0 |
| 9 | 0xFDFF | Q9=0 |
| 11 (aucun) | 0xF7FF | Tous à 1 → tous éteints |

!!! info "Plan 11 = Anti-ghosting"
    `select_plane(11)` envoie `~(1<<11) = 0xF7FF`, ce qui met **toutes les 10 sorties Q0–Q9 à 1**, bloquant tous les MOSFET. C'est utilisé entre deux plans pour éviter le ghosting.

## Timing STCP

L'impulsion STCP (Storage Clock Pulse) transfère les données du shift register vers le latch output :

```
SPI transfert → données dans shift register
STCP ↑         → transfert shift register → output latch
STCP ↓         → MOSFET commute selon nouvelles données
```

La durée d'impulsion de **1 µs** est largement suffisante pour le 74HC595 (t_setup min = 25 ns @ 5V).
